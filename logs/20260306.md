# 作業ログ 2026-03-06

## 仕様決定

### Claude Code 自動化環境の初期構築
- CLAUDE.md をプロジェクトのルールファイルとして採用
  - プロジェクト概要・厳守ルール・作業ディレクトリ・開発対象を記載
- logs/YYYYMMDD.md を作業ログの保存形式として採用
- context.md をセッション間コンテキスト引き継ぎファイルとして採用
  - CLAUDE.md・最新ログ・ファイル構成を集約する形式

### 運用ルールの確定
- ログ記録対象を「仕様決定・仕様変更・未解決課題」の3つに限定
- 起動時に logs/ 最新ファイルを読み込み、前回の作業内容を冒頭で報告するルールを採用
- セッション終了時に抽出→ログ記録→context.md更新→pushを行うフローを採用
- claude.ai 新スレッド開始時は CLAUDE.md と最新ログの raw URL を貼り付けて参照する運用を採用

## 仕様変更

### ログ記録セクションの再定義
- 変更前: `やり取り記録`（「logs/YYYYMMDD.md に追記すること」「GitHub に push すること」の2行のみ）
- 変更後: `ログ記録ルール`（記録する3項目と記録しないものを明示）
- 変更理由: ログの粒度を統一し、不要な情報を排除するため
